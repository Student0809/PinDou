# AI拼豆库存管理 - 设计文档

## 概述

一个移动端PWA网页应用，用于管理拼豆库存，支持入库、消耗、AI识别和统计功能。

## 技术栈

| 技术 | 用途 |
|------|------|
| Vue 3 | 前端框架 |
| Vite | 构建工具 |
| Vant 4 | 移动端UI组件库 |
| Pinia | 状态管理 |
| localStorage | 数据持久化 |
| PWA | 离线支持、添加到桌面 |

## 架构设计

```
┌─────────────────────────────────────┐
│            AI拼豆 (PWA)              │
├─────────────────────────────────────┤
│  Vue 3 + Vite + Vant 4 + Pinia      │
├─────────────────────────────────────┤
│  localStorage (数据持久化)           │
└─────────────────────────────────────┘
```

### 页面结构

采用底部Tab导航，共4个页面：

| Tab | 页面 | 功能 |
|-----|------|------|
| 库存 | 库存页面 | 显示库存列表、统计、告急提醒 |
| 入库 | 入库页面 | 添加豆子到库存 |
| 消耗 | 消耗页面 | 从库存消耗豆子、AI识别 |
| 设置 | 设置页面 | 初始化库存、AI配置、记录查看 |

## 数据模型

### 库存数据 (Inventory)

```typescript
interface Inventory {
  colorCode: string;    // 色号，如 "A1"
  quantity: number;     // 数量
}
```

### 操作记录 (Record)

```typescript
interface Record {
  id: string;
  type: 'add' | 'consume';  // 入库/消耗
  colorCode: string;        // 色号
  quantity: number;         // 数量
  timestamp: number;        // 时间戳
  note?: string;            // 备注（如"AI识别"）
}
```

### AI配置 (AIConfig)

```typescript
interface AIConfig {
  apiUrl: string;      // API地址
  apiKey: string;      // API密钥
  modelName: string;   // 模型名称
}
```

### 设置 (Settings)

```typescript
interface Settings {
  alertThreshold: number;  // 告急阈值，默认300
}
```

### 存储结构

```typescript
interface Storage {
  inventory: Inventory[];     // 库存列表
  records: Record[];          // 操作记录
  aiConfig: AIConfig | null;  // AI配置（可选）
  settings: Settings;         // 设置
}
```

**功能列表：**
- 显示所有色号的库存列表（色号 + 数量）
- 告急标识：数量 < 告急阈值 标红并置顶
- 排序功能：按数量升序/降序、按色号排序
- 筛选功能：按色系筛选（A系、B系...）
- 顶部统计：显示总数量、各色系小计（可折叠）
- 点击列表项可快速跳转到入库/消耗

## 核心流程

### 初始化流程

```
用户打开应用
  → 检测localStorage是否有库存数据
  → 无数据：显示欢迎页，提示去设置页初始化
  → 有数据：加载并显示库存列表
```

### 入库/消耗流程

```
选择色号 + 输入数量
  → 点击确认
  → 校验数量（消耗时检查是否超出库存）
  → 更新库存数据
  → 记录操作日志
  → 显示成功提示
  → 刷新列表
```

### AI识别流程

```
点击AI识别按钮
  → 打开图片选择器（从相册选择）
  → 进入图片裁剪界面（必须裁剪）
  → 调用AI API识别
  → 显示识别结果（如"A1 100, B2 50"）
  → 用户确认或修改结果
  → 执行消耗操作
```

### 告急逻辑

```
用户在设置页配置告急阈值（默认300）

库存列表渲染时：
  → 过滤出 quantity < 告急阈值 的项
  → 标红并排在列表最前
  → 其余按当前排序规则显示
```

## 技术要点

### PWA配置
- 使用 `vite-plugin-pwa` 插件
- 支持离线访问
- 支持添加到桌面

### 图片裁剪
- 使用 `cropperjs` 库实现图片裁剪功能
- 移动端触摸友好

### AI API调用
- 支持OpenAI兼容格式的API
- 使用Vision能力识别图片中的色号和数量
- Prompt模板：识别图片中的拼豆色号和数量，输出格式如"A1 100, B2 50"
